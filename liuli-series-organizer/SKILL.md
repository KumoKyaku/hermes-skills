# 琉璃神社 按系列整理工具

## 适用场景
琉璃神社目录结构为 `YYYY年合集/YYYY年MM月合集/{CHS,CHT,RAW,XX月海报}/`，需要用硬链接按视频系列重新组织目录。

**基础路径：** `E:\SystemCacheInfo\琉璃神社\`
**输出路径：** `E:\SystemCacheInfo\琉璃神社_按系列\`

## 目录结构

### 原始目录
| 年份范围 | 月文件夹名 | 视频子目录 | 海报位置 |
|---------|-----------|-----------|---------|
| 2019-2020 | `[桜都字幕组]YYYY年MM月合集` | BIG5, GB, RAW | 封面/ |
| 2022-2026 | `YYYY年MM月合集` | CHS, CHT, RAW | XX月海报/ |

另有 `前作/` 目录存放历史集数（需要去重处理）。

### 输出目录
```
琉璃神社_按系列/
  [Studio]Series/
    CHS/ → .chs.mp4 硬链接
    CHT/ → .cht.mp4 硬链接
    RAW/ → .mkv 硬链接
    海报/ → .png 硬链接
    封面/ → .jpg 硬链接
    简评.txt → 从月度简评提取
```

## 规则

### 系列名提取
- 去掉日期前缀 `[YYMMDD]` 或 `[桜都字幕组][YYMMDD]`
- 去掉扩展名，去掉话数后缀（`＃X` `第X話` `第X巻` `前編` `後編` 等）
- Studio 名称去前后空格
- **归一化：** 去掉 `OVA` 前缀和末尾 `THE ANIMATION`，使 `OVA XXX` 和 `XXX` 合并为一系
- **标点归一：** `!` → `！`、`?` → `？`（全角/半角统一），防止同一系列因标点全半角差异分家

### 单集跳过
如果一个系列所有文件只对应同一话 → 单集作品，不创建目录

### 硬链接
- 使用 `os.link()` 创建 NTFS 硬链接（不占额外空间）
- 同系列同话数同语言下，按 (ep_num, lang) 去重，只保留第一个
- **前作目录** 扫描时用 dedup_check=True，跳过已有组合

### 前作去重
月度目录下的 `前作/` 目录存放历史集数（与 CHS/CHT/RAW 中的文件重复）：
- 扫描前作中的所有视频和媒体文件
- 按 (ep_num, lang) 组合去重：如果某个话数+语言的视频已从 CHS/CHT/RAW 目录收录，前作中的同话数+语言文件自动跳过
- 前作中的海报/封面也进行匹配，但同名文件去重

### 海报/封面匹配
- 去掉 `[封面]` 前缀，去掉话数后缀后匹配系列名
- 先精确匹配，再模糊匹配（共享文字）

### 简评生成
- 解析 `*年*月里番简评.txt` 中的 `《作品名》` 条目
- 按作品名匹配到系列
- 每个系列生成 `简评.txt`，汇总所有相关月份评论

### 忽略内容
- `*月预告/` 目录
- `.zip` 文件

## 执行脚本位置
- 分析脚本：`E:\SystemCacheInfo\琉璃神社\analyze_series.py`
- 执行脚本：`E:\SystemCacheInfo\琉璃神社\execute_series_v2.py`
- 分析结果：`E:\SystemCacheInfo\琉璃神社\series_analysis.json`

## 注意事项与陷阱
- 硬链接必须在同一 NTFS 卷
- 日文字符（Unicode）正确处理 — Python `os.link()` 和 `pathlib` 支持 Unicode 路径
- 原目录结构不变，不要删除任何原始文件
- 执行前先跑分析扫描，确认报告后再执行硬链接
- **2019-2020 格式差异：** 存在 `[720P BIG5]`（同一括号）和 `[720P][BIG5]`（分开括号）两种格式，正则需要同时处理
- **Studio 空格：** 部分文件名中有 `[ King Bee]`（前导空格），需要 strip()
- **标点陷阱：** `!` 和 `！`、`?` 和 `？` 在日文文件名中混用，必须在归一化时统一为全角
- **前作重复：** `前作/` 目录可能包含 200+ 个重复文件，必须按 (ep_num, lang) 去重
- **海报 JPG 重复：** 同一张海报可能同时出现在 `XX月海报/` 和 `前作/` 中，硬链接时按文件名去重
- 硬链接必须在同一 NTFS 卷
- 日文字符（Unicode）正确处理
- 原目录结构不变
- 执行前确认分析报告
